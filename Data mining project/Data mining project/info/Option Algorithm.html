Instruction

  • Choose one or at most two challenging data mining algorithms that you'd like to implement yourself.

You could for example consider implementing an extension, suggested in the literature, of one of the algorithms discussed in the course. Basic versions of algorithms considered in the course are not recommended.

  • Implement them and apply them to a suitable data set.
  • Write a report following the guidelines below.
Feel free to use any language (Python, C++, Java, Matlab...) that you like.
 
You do not necessarily have to write the software from scratch, but may instead extend existing software, see the software links. Do indicate that in your proposal! Plagiarism of any kind is of course prohibited and will result in a failure for this course.
 
Proposal guidelines
 
The proposal can be half a page. It should contain the following items.
  • A short description of the algorithm(s) you want to use or implement.
  • A description of or link to the data set that you will illustrate your algorithm on.
  • Pointers to the literature that you use for inspiration and comparison.
Below you can find some suggestions for data sets and software. If you're up to a real challenge, you can enter one of the data mining or machine learning competitions.
 
Report guidelines
 
The report should read like a scientific paper of about 6 to 8 pages, containing the following items.
  • A descriptive title.
  • An abstract (about 100 words) that summarizes the research problem, approach/method, and main results.
  • In the body of the report, you should explain clearly, not necessarily in the following order:
  1. a mathematical description of the algorithm and an analysis of the computational complexity of the algorithm, i.e., how it scales as a function of the number of records and features;
  2. related previous work about the same or similar algorithms;
  3. the type of problems the algorithm tries to solve;
  4. a description of the illustration on a real-world data set;
  5. the main results of your project;
  6. an analysis of results and evaluation of your project outcome: to what extent it matches your expectation and why if there is a gap;
  7. possible future directions to improve the results;
  8. give us insight into your code, e.g., by adding and explaining the most important parts in an appendix, or by adding a link to a github page.
Remember that it is important not only to tell what you did, but also to motivate why you made those choices. Also clearly identify any innovative ideas that you explored in your project, by comparing related work to the best of your knowledge.
 
Criteria for evaluation
 
The report will be evaluated based on the following criteria.
  • Understanding of material: does it show that you know what you are talking about?
  • Clarity of exposition: is the report easy to read and understand?
  • Correctness: are there any major flaws?
  • Completeness: did you address all the points listed above?
  • Originality: does the project contain novel, innovative ideas?
